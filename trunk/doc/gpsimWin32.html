gpsimWin32

gpsim native WIN32 port

gpsim - The gnupic simulator, a simulator for Microchip PIC microcontrollers.

Introduction

This document describes how to compile and install gpsim on WIN32 platforms.

Gpsim can be compiled in a CLI - command line interface mode (no gui), or in GUI - graphical user interface mode.
 

Which tools are required, where to get them

Microsoft Visual Studio .NET 2003 is required to build WIN32 port of gpsim.

MinGW gcc or Cygwin gcc -mno-cygwin compiler is required for WIN32 native compilation of additional packages.

Some additional GNU tools are required to build gpsim:

  • bison (or some other yacc compatible parser generator)
  • flex (or some other lex compatible lexical analyzer generator)

The recommended installation for bison and flex is from Cygwin environment. The other possibility is to use nativeWIN32 ports of bison and flex, which are part of UnxUtils project and GnuWin32 project.

Cygwin environment is required (or at least recommended) for WIN32 native compilation of additional packages.
 

How to set up development environment for WIN32 port of gpsim

Cygwin installation:

Select DOS as the Default Text File type. In the Devel Category select bison and flex.
You have to have c:\cygwin\bin specified in the PATH environment variable, but I think that this is done by default from the Cygwin installation.

In Microsoft Visual Studio .NET 2003 select Tools/Options...
In the Options dialog select Projects VC++ Directories and add C:\cygwin\bin path.
 

Which additional packages are required, where to get them, how to to install them

CLI:

The CLI WIN32 version of gpsim requires 2 additional packages: popt and glib.

  • popt is a command line option parsing library. The following packages, which can be downloaded from GnuWin32 project, are required:
    • popt-1.8-1-lib.zip: required for gpsim development
    • popt-1.8-1-bin.zip: required by gpsim run time
    • libintl-0.11.5-2-bin.zip: required by popt run time
    • libiconv-1.8-1-bin.zip: required by libintl run time

  • glib is the low-level core library that forms the basis for projects such as GTK+ and GNOME.
    The WIN32 port of glib package is a part of Tor Lillqvist's GTK+ (and GIMP) for Windows project. The latest version of developer, runtime and source package can be found at Tor Lillqvist--GTK+ (and GIMP) for Windows--Downloads.
    • glib-dev-2.4.2.zip: required for gpsim development
    • glib-2.4.2.zip: required by gpsim GUI run time

GUI:

The GUI WIN32 version of gpsim requires the following additional packages, in addition to popt and glib, mentioned in the CLI section:

  • readline - The Readline library provides a set of functions for use by applications that allow users to edit command lines as they are typed in.
    • readline-4.3-5-mswin32.zip: sources

  • gtkextra-2 - GtkExtra is a useful set of widgets for creating GUI's for the Xwindows system using GTK+. You can use it complementary to GTK+ and it is written in C. It is also Free Software and released under the LGPL license.
    • gtk+extra-2-src-1.1.0-20040626.tar.gz: sources from CVS 11th June 2004, with the patch applied. Sources from the package can be successfully compiled on WIN32 using Cygwin gcc -mno-cygwin native compilation and on Red Hat Linux 9.0.
    • gtk+extra-2-dev-1.1.0-20040626.zip: header files and static library, built from gtk+extra-2-src-1.1.0-20040626.tar.gz


The WIN32 ports of following packages are a part of Tor Lillqvist's GTK+ (and GIMP) for Windows project. The latest version of developer, runtime and source packages can be found at Tor Lillqvist--GTK+ (and GIMP) for Windows--Downloads.

  • gtk - GTK+ is a multi-platform toolkit for creating graphical user interfaces.
    • gtk+-dev-2.4.3.zip: required for gpsim development
    • gtk+-2.4.3.zip: required by gpsim GUI run time
    • gettext-runtime-0.13.1.zip: required by GTK+ run time
    • libiconv-1.9.1.bin.woe32.zip: required by GTK+ run time

  • pango - Pango is a library for layout and rendering of text, with an emphasis on internationalization. It forms the core of text and font handling for GTK+-2.0.
    • pango-dev-1.4.0.zip: required for gpsim development
    • pango-1.4.0.zip: required by gpsim GUI run time

  • atk - The ATK library provides a set of interfaces for accessibility.
    • atk-dev-1.6.0.zip: required for gpsim development
    • atk-1.6.0.zip: required by gpsim GUI run time
       

Setting up the development environment

  • Create the gpsim_proj directory for the complete gpsim project
  • Create the gpsim subdirectory for the gpsim package and download the latest gpsim version from CVS

CLI:

  • Create the readline subdirectory and extract readline-4.3-5-mswin32.zip into it
  • Create the popt subdirectory and extract popt-1.8-1-lib.zip into it
  • Create the glib subdirectory and extract glib-dev-2.4.2.zip into it
    • For the development of CLI mode, only header files from glib package are required. The only functionality needed from the library for the command line mode is slist (Singly-Linked Lists), which is currently replaced with the implementation in plat/win32/glist.cpp.
    • For the development of GUI mode, the complete glib library is required.
  • Double-click on gpsim.sln to start the Microsoft Visual Studio .NET 2003, select Debug or Release solution configuration and rebuild the solution.

GUI:

  • Execute steps, described in previous CLI section
  • Create the gtk+ subdirectory and extract gtk+-dev-2.4.3.zip into it
  • Create the pango subdirectory and extract pango-dev-1.4.0.zip into it
  • Create the atk subdirectory and extract atk-dev-1.6.0.zip into it
  • Use precompiled gtk+extra-2-dev-1.1.0-20040626.zip:
    • Create the gtkextra-2 subdirectory and extract gtk+extra-2-dev-1.1.0-20040626.zip into it
    or compile gtk+extra-2-src-1.1.0-20040626.tar.gz:
    • Create the gtkextra-2 subdirectory and extract the content of gtk+extra-1.1.0 directory from gtk+extra-2-src-1.1.0-20040626.tar.gz into it
    • Go to gtkextra-2\gtkextra directory, run bash and execute
      make -f Makefile.win32
      create gtkextra-2\lib directory and copy gtkextra-2\gtkextra\libgtkextra-win32-2.0.a to gtkextra-2\lib\libgtkextra-win32-2.0.lib
  • Double-click on gpsim.sln to start the Microsoft Visual Studio .NET 2003, select Debug GUI or Release GUI solution configuration and rebuild the solution.
     

Setting up the runtimee

CLI:

  • Extract popt1.dll from popt-1.8-1-bin.zip package, bin directory o the same directory where gpsim.exe resides.
  • Extract  libintl-2.dll from libintl-0.11.5-2-bin.zip package, bin directory to the same directory where gpsim.exe resides.
  • Extract  libiconv-2.dll from libiconv-1.8-1-bin.zip package, bin directory to the same directory where gpsim.exe resides.

GUI:

  • Extract bin\libglib-2.0-0.dll, bin\libgobject-2.0-0.dll and bin\libgmodule-2.0-0.dll from glib-2.4.2.zip package to the same directory where gpsim.exe resides.
  • Extract bin\iconv.dll from libiconv-1.9.1.bin.woe32.zip package to the same directory where gpsim.exe resides.
  • Extract bin\intl.dll from gettext-runtime-0.13.1.zip package to the same directory where gpsim.exe resides.
  • Extract everything in bin directory, complete etc and complete lib directory from gtk+-2.4.3.zip package to the same directory where gpsim.exe resides.
  • Extract bin\libpango-1.0-0.dll and bin\libpangowin32-1.0-0.dll, complete etc and complete lib directory from pango-1.4.0.zip package to the same directory where gpsim.exe resides.
  • Extract bin\libatk-1.0-0.dll from atk-1.6.0.zippackage to the same directory where gpsim.exe resides.
     

gpsimWin32 snapshots

These binaries are occasionally built from the developmental code in the CVS server. They are not official releases and should be considered "beta" quality. Some may not even work at all if a serious bug was introduced during development, so several are provided to give you the best chance of finding one that will work.
